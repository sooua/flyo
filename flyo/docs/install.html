Install Flyo

Flyo is a single binary. The whole install process is "download, extract, run".

Download

Grab the latest release for your platform from the releases page. File names follow this convention:

flyo-<version>-<target>.<ext>

# examples
flyo-0.1-x86_64-pc-windows-msvc.zip
flyo-0.1-x86_64-unknown-linux-musl.tar.gz
flyo-0.1-aarch64-unknown-linux-musl.tar.gz
flyo-0.1-aarch64-apple-darwin.tar.gz
flyo-0.1-mipsel-unknown-linux-musl.tar.gz   # OpenWrt

Windows (10 / 11)

  1. Download flyo-0.1-x86_64-pc-windows-msvc.zip.
  2. Right-click → Extract All to a folder, e.g. D:\flyo\.
  3. Open the folder, edit webd.conf in Notepad — set Webd.Root to the path you want to share and pick at least one Webd.User.
  4. Double-click flyo.exe to launch. A console window opens; closing it stops the server.
  5. Open http://127.0.0.1:9212 in your browser.

Linux (x86_64 / ARM64)

tar xzf flyo-0.1-x86_64-unknown-linux-musl.tar.gz
sudo mv flyo /usr/local/bin/

# Create a working directory with a config
mkdir -p ~/flyo && cd ~/flyo
cat > webd.conf <<EOF
Webd.Root   /mnt/data
Webd.Listen 9212
Webd.User   rlumS admin secret
Webd.Guest  rl
EOF

# Run
flyo

For ARM64 boards (Raspberry Pi 4, Orange Pi 5, Apple servers etc.) use the aarch64-unknown-linux-musl build instead.

macOS (Apple Silicon)

tar xzf flyo-0.1-aarch64-apple-darwin.tar.gz
sudo mv flyo /usr/local/bin/

# First launch — Gatekeeper will quarantine the binary
xattr -d com.apple.quarantine /usr/local/bin/flyo

flyo -w ~/Downloads -u rlumS:admin:secret

OpenWrt (mipsel / ARM routers)

  1. SSH into your router. Check architecture: cat /etc/openwrt_release.
  2. Grab the matching tarball (look for mipsel, mips, or aarch64 with musl).
  3. scp the binary onto the router, then:
chmod +x /usr/bin/flyo

# Persist start command
cat > /etc/init.d/flyo <<'EOF'
#!/bin/sh /etc/rc.common
START=99
start() { /usr/bin/flyo -w /mnt/sda1 -u rlumS:admin:secret & }
stop()  { killall flyo; }
EOF
chmod +x /etc/init.d/flyo
/etc/init.d/flyo enable
/etc/init.d/flyo start

Android (via adb)

Useful for turning a spare phone into a tiny file server on your home network.

adb push flyo /data/local/tmp/
adb shell
cd /data/local/tmp && chmod 755 flyo
mkdir -p /sdcard/.Trash
nohup ./flyo -w /sdcard -u rlumS:admin:secret >/dev/null 2>&1 &
ifconfig   # note the wifi IP

First run

Out of the box, with no config file, Flyo serves the current working directory on 0.0.0.0:9212 with guest read-only access. Any of these will work:

flyo                                        # use ./webd.conf or defaults
flyo -w /path/to/share                      # CLI override of share root
flyo -w /mnt/data -u rlumS:admin:secret     # ad-hoc admin user

Then point a browser at http://your-host:9212.

Auto-start

Flyo doesn't bring its own service manager — use whatever your OS already has.

systemd (most Linux distros)

cat > /etc/systemd/system/flyo.service <<EOF
[Unit]
Description=Flyo file server
After=network.target

[Service]
ExecStart=/usr/local/bin/flyo
WorkingDirectory=/etc/flyo
Restart=on-failure
User=flyo

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable --now flyo

Windows — Task Scheduler

  1. Open Task Scheduler → Create Basic Task.
  2. Trigger: When the computer starts.
  3. Action: Start a program → C:\flyo\flyo.exe, Start in: C:\flyo.
  4. Check Run whether user is logged on or not.
