Configuration

Flyo reads a plain-text config file. The format is intentionally identical to the original webd.conf — drop your existing config in and it works.

Config lookup order

On startup, Flyo tries these paths in order. The first one that exists is loaded.

  1. ./flyo.conf
  2. ./webd.conf
  3. /etc/flyo.conf
  4. /etc/webd.conf

If none exist, Flyo serves the current working directory on 0.0.0.0:9212 with the default guest-read policy.

Directives

Directives are case-insensitive. Quoted strings let you use paths with spaces. Lines starting with # are comments.

DirectiveArgumentsPurpose
Webd.Root <path> The folder you want to share. Quote it if it contains spaces.
Webd.Listen <port> or <addr:port> Where to listen. Repeatable. 9212 → all interfaces. [::]:9212 → dual-stack.
Webd.User <tag> <name> <pass> Add a user with a permission tag. Max 3 users.
Webd.Guest <tag> or 0 Permissions for anonymous visitors. 0 disables guest entirely.
Webd.Hide — Windows: don't show the tray icon.
Webd.Browser "<path>" Windows: which browser to launch on tray double-click.

Permission tags

Each user (and the guest) gets a string of tag characters. Each tag is independent — combine freely.

TagMeaningNotes
rAccess (download) filesRequired to view file contents.
lList directoriesWithout this, the folder appears empty.
uUpload + create folderDrag-drop, paste, mkdir.
mDelete / move / renameDeletes go to .Trash/, not real removal.
SShow hidden filesDot-prefixed entries become visible.
TUse media player pageActivates the /.player.htm viewer for video/audio.
DForce downloadAdds Content-Disposition: attachment instead of inline display.

Use 0 as the tag to mean "no permissions" — only useful for Webd.Guest 0 to disable anonymous access.

Examples

Home NAS — me writes, family reads

Webd.Root   /mnt/data/share
Webd.Listen [::]:9212

# I get everything
Webd.User   rlumST admin    correct-horse-battery-staple

# Family gets read + streaming
Webd.User   rlT    family   shared-passphrase

# Strangers see nothing
Webd.Guest  0

Public drop box — anyone can upload, no one can list

Webd.Root   /mnt/dropbox
Webd.Listen 9212

# Upload-only guest — they can drop files but can't see what's inside
Webd.Guest  u

# I can clean it out
Webd.User   rlum admin secret

Router / OpenWrt — strict read-only mirror

Webd.Root   /mnt/sda1/public
Webd.Listen 9212

Webd.Guest  rl
# no users → no writes ever possible

CLI override

Every directive has a short flag for quick ad-hoc serving.

FlagEquivalent
-w <path> Webd.Root
-l <addr:port>Webd.Listen
-u tag:user:pass Webd.User
-g tag Webd.Guest
-c <path> Use a specific config file
 
# One-line server, no config file needed
flyo -w /tmp/incoming -u rlumS:admin:secret -g 0

Environment variables

For OS-level concerns that don't belong in the config file.

VariablePurpose
RUST_LOG Log level. info (default), debug, warn, etc.
_LOG_DIR Prefix for daily rotated log files (Linux).
_RUNAS Drop privileges to this user after binding the port (Linux).
_CHROOT_PATHchroot into this directory after startup (Linux).
