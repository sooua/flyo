The file server you actually want to host yourself.

Flyo is a tiny self-hosted file sharing server written in Rust. Drop one binary on any box — Windows, Linux, ARM NAS, OpenWrt router — and a folder is on the web with a modern UI in seconds.

↓ Download & install See it work
~/sharedata $ flyo 
# A complete file server in 3 lines.
$ ./flyo -w /mnt/media -u rlumS:admin:secret
INFO  loading config from webd.conf
INFO  flyo listening on http://0.0.0.0:9212
# Open http://your-ip:9212 — modern UI, dark mode, drag-drop, Range streaming.

Runs natively on

Windows 10+ Linux x64 / ARM64 macOS OpenWrt Armbian Android (adb)

Small enough for a router. Capable enough for a NAS.

Every feature you'd want from a self-hosted file UI, with none of the bloat. No database, no Docker, no Electron.

⬇

2 MB single binary

Drop and run. Zero dependencies, zero install steps, zero background services to manage.

🛡

Memory-safe Rust

The HTTP stack is built on the same library Cloudflare and AWS use. No buffer overflows by design.

🌓

Modern, dark-mode UI

A clean Cal-Sans-and-Inter interface that respects your system theme. Keyboard, mobile, screen-reader friendly.

⇅

Drag-and-drop uploads

Drop files, folders, or pasted screenshots anywhere on the page. Atomic writes — no half-uploaded files visible.

▶

Range streaming

Scrub videos directly from the browser. Resume large downloads. HTTP Range works out of the box.

🔑

Per-user permissions

Seven independent tags — read, list, upload, modify, show-hidden, media-play, force-download. Up to 3 users plus guest.

⚡

HTTP/2 + gzip

Multiplexed connections and compressed responses by default. Listings load instantly even over slow links.

📝

Drop-in webd config

Already use webd? Point flyo at your existing webd.conf — same directives, same semantics. Zero migration.
